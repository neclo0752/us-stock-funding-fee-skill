# us-stock-funding-fee-skill

该skill主要用于比较“汇丰香港入金嘉信”5种路径手续费，计算出最省钱入金方式。
当用户输入“多少美元/多少港币”时，输出 5 个方案的手续费、最低成本方案和关键临界点。

## 功能

- 支持输入币种：`USD` / `HKD`
- 支持可选汇率参数：`USDHKD`（默认 `1 USD = 7.8 HKD`）
- 固定输出 5 个方案手续费对比（不省略）
- 自动给出手续费最低方案
- 自动补充关键临界点（方案一 vs 方案二）

## 5种方案定义

| 方案 | 路径 | 手续费规则 | 备注 |
|---|---|---|---|
| 方案一 | 汇丰香港 -> 嘉信花旗香港 | `0.16%` | 嘉信端港币兑美元折损 |
| 方案二 | 汇丰香港 -> 嘉信花旗美国 | `21.88 USD` 固定 | 固定手续费 |
| 方案三 | 汇丰香港 -> wise(换美元) -> 嘉信花旗美国 | `0.4%` | wise按比例收兑换费 |
| 方案四 | 汇丰香港 -> wise -> 嘉信花旗美国 | `10 HKD` 固定 | 需wise港币账户 |
| 方案五 | 汇丰香港 -> 汇丰美国 -> 嘉信花旗美国 | `0` | 需HSBC US账户 |

## 计算规则

详见 [references/calculation-rules.md](references/calculation-rules.md)。

核心变量：
- `A`: 输入金额
- `C`: 输入币种（USD/HKD）
- `R`: 汇率（USDHKD，默认 7.8）

关键临界点：
- 方案一 vs 方案二：`21.88 / 0.0016 = 13,675 USD`
- 折算成人民币约：`92,574 CNY`（按 `1 USD = 6.768 CNY`）
- Q5双换汇临界点：约 `3,911 USD`

## 使用示例

示例输入：
- `我有8000美元，汇丰香港入金嘉信5种方案手续费分别是多少？`
- `我有20万港币，5种方案哪个手续费最低？`
- `10万港币，按1美元=7.82港币算一下5种方案手续费`

示例输出结构：
1. 复述输入（金额、币种、汇率）
2. 5方案手续费对比表（主币种 + 括号内换算）
3. 最低手续费方案
4. 临界点与风险提示（wise路径稳定性/合规追溯）

## 注意事项

- 本 skill 只比较“手续费成本”，不把汇率波动收益当作手续费优势。
- 若未提供币种，应先补齐币种后再计算。
- 对大额资金，优先考虑传统银行通道的稳定性与合规可追溯性。
